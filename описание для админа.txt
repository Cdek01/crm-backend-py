Вот подробная инструкция, описывающая, что и как вы, как администратор, можете делать.
Обзор возможностей админ-панели

Ваша админ-панель теперь позволяет управлять тремя ключевыми уровнями доступа:
Суперадминистратор: Полный доступ ко всем данным всех клиентов. Этот доступ выдается вручную и предназначен только для владельцев системы.
Роли: Группы прав внутри одного клиента (например, "Менеджер", "Бухгалтер").
Разрешения: Конкретные права на действия (например, "просматривать лидов", "редактировать кастомную таблицу 'Проекты'").
Задача 1: Создание нового клиента и пользователя
Прежде чем настраивать роли, вам нужны те, для кого вы их настраиваете.

1. Создание Клиента (Тенанта)
Где: Меню слева -> "Клиенты".
Что делать: Нажмите "Create", введите название компании-клиента (например, "ООО Ромашка") и нажмите "Save".

2. Создание Пользователя
Где: Меню слева -> "Пользователи".
Что делать:
Нажмите "Create".
Email / Full Name: Введите данные сотрудника.
Tenant: Из выпадающего списка выберите клиента, к которому принадлежит этот сотрудник (например, "ООО Ромашка").
Is Superuser: Оставляйте этот чекбокс пустым! Он предназначен только для системных администраторов.
Roles: Пока оставьте это поле пустым. Мы назначим роль на следующем шаге.
Нажмите "Save". Помните: пароль для этого пользователя нужно будет задать через API или напрямую в базе данных, так как мы убрали это поле из админки.
Задача 2: Настройка доступа для "Менеджера по продажам"
Цель: Создать роль, которая позволяет сотрудникам только просматривать и создавать новых лидов, а также просматривать физ. лиц.

1. Создание Роли
Где: Меню слева -> "Роли".
Что делать:
Нажмите "Create".
Name: Введите Менеджер по продажам.
Tenant: Выберите клиента ("ООО Ромашка").
Permissions: Перед вами будет список всех возможных действий в системе. Зажмите Ctrl (или Cmd на Mac) и выберите следующие права:
leads:view (Просмотр списка лидов)
leads:create (Создание новых лидов)
individuals:view (Просмотр списка физических лиц)
Нажмите "Save".

2. Назначение Роли Пользователю
Где: Меню слева -> "Пользователи".
Что делать:
Найдите пользователя, которого вы создали ранее, и нажмите на иконку редактирования (карандаш).
Пролистайте до поля "Roles".
Зажмите Ctrl (или Cmd) и выберите в списке только что созданную роль Менеджер по продажам.
Нажмите "Save".
Результат: Теперь этот пользователь, войдя в систему, сможет получать список лидов и создавать новых, но при попытке отредактировать или удалить лида он получит ошибку "403 Forbidden".

Задача 3: Настройка доступа к кастомной таблице
Цель: Клиент создал через API кастомную таблицу "Договоры" (contracts). Нужно дать юристу право просматривать и редактировать записи в этой таблице.

1. Создание кастомной таблицы (делается через API)
Предположим, ваш клиент уже сделал POST запрос на /api/meta/entity-types и создал таблицу с name: "contracts". В этот момент ваш бэкенд автоматически создал в базе новые разрешения: data:view:contracts, data:edit:contracts и т.д.

2. Создание Роли "Юрист" в админ-панели
Где: Меню слева -> "Роли".
Что делать:
Нажмите "Create".
Name: Введите Юрист.
Tenant: Выберите нужного клиента.
Permissions: Пролистайте список. Вы увидите там новые, только что появившиеся разрешения. Зажмите Ctrl и выберите:
data:view:contracts
data:edit:contracts
Нажмите "Save".

3. Назначение Роли Пользователю-Юристу
Где: Меню слева -> "Пользователи".
Что делать:
Откройте карточку пользователя-юриста на редактирование.
В поле "Roles" выберите роль Юрист.
Нажмите "Save".
Результат: Теперь юрист сможет через API запрашивать и обновлять данные по URL /api/data/contracts, но не сможет, например, удалить запись или посмотреть данные из другой кастомной таблицы, к которой ему не дали доступ.
